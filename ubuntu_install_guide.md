# Ubuntu 24.04 LTS サーバー インストール詳細手順

このドキュメントは、ベアメタルサーバーに Ubuntu 24.04 LTS (Server) をインストールするための詳細な手順書です。
`ubuntu_setup_flow.md` の §1.1 の詳細版に相当します。

---

## 前提: 用意するもの

| 項目 | 詳細 |
|------|------|
| USBメモリ | 8GB以上（内容は上書きされます） |
| 作業PC | ISOを書き込むためのWindows/Mac/Linux機 |
| 有線LANケーブル | インストール中のパッケージ取得に使用 |
| モニター & キーボード | インストール作業用（一時的に接続） |

> **⚠️ ハードウェア固有の注意 (Core i5-12600KF):**
> このCPUには内蔵グラフィックス (iGPU) が搭載されていません（末尾が `F` のモデル）。そのため、**モニターのケーブルはマザーボードの映像出力端子ではなく、必ず RTX 3070 Ti のグラフィックボードの端子に直接接続**してください。マザーボードに繋ぐと画面に何も表示されません。

---

## Step 1: Ubuntu Server ISOのダウンロード

1. 公式ダウンロードページにアクセスします。
   - **URL:** https://ubuntu.com/download/server
2. 「Ubuntu Server 24.04 LTS」を選択し、ISOファイルをダウンロードします。
   - ファイル名の例: `ubuntu-24.04.2-live-server-amd64.iso`

> **⚠️ 重要:** LTS（Long Term Support）版を選択してください。数字だけのバージョン（非LTS）は保守期間が短いです。

---

## Step 2: 起動可能なUSBメモリの作成

### Windows の場合 (Rufus を使用 ← 推奨)

1. [Rufus](https://rufus.ie/ja/) をダウンロードして起動します。
2. 以下の通り設定します。

   | 項目 | 設定値 |
   |------|--------|
   | デバイス | 対象のUSBメモリを選択 |
   | ブートの種類 | ダウンロードしたISOファイルを選択 |
   | パーティション構成 | **GPT** |
   | ターゲットシステム | **UEFI (CSM なし)** |
   | ファイルシステム | Large FAT32 (規定値のまま) |

3. 「スタート」をクリックし、警告ダイアログが出たら「OK」します。
4. 書き込み完了まで待ちます（数分程度）。

---

## Step 3: BIOSの設定（起動順序の変更）

1. サーバーにモニターとキーボードを接続し、電源を入れます。
2. 起動直後に **Delete キー** または **F2 キー** を連打してBIOS/UEFI設定画面に入ります。
   - マザーボードによってキーが異なります（`F2`, `F10`, `F12`, `DEL` など）
3. **Boot** メニューで、起動順序 (Boot Priority) を **USBメモリが1番**になるよう変更します。
4. **Secure Boot (セキュアブート)** は、必ず **Disabled (無効)** に変更してください。
   - **理由:** OSインストール後に RTX 3070 Ti 用の NVIDIA プロプライエタリ・ドライバをインストールする際、セキュアブートが有効だと署名検証（MOKの登録作業）という非常に煩雑な手順が発生し、失敗するとドライバがロードされずGPUが使えなくなってしまいます。
5. 設定を保存して再起動します（通常は `F10` で保存＆終了）。

---

## Step 4: インストーラーの起動

1. USBメモリを挿入した状態でサーバーを起動します。
2. しばらく待つと、紫色のGRUBメニューが表示されます。
3. 「**Try or Install Ubuntu Server**」を選択して `Enter` を押します。
4. カーネルの読み込みが始まり、インストーラーが起動します（1〜2分かかる場合があります）。

---

## Step 5: インストール設定（詳細）

### 5.1 言語選択
- インストーラーの言語を選択します。
- ここでは **English** を推奨します（サーバー用途はエラーログ等が英語のため）。
- `Enter` で決定します。

### 5.2 キーボードレイアウト
- 「Keyboard configuration」画面が表示されます。
- 日本語キーボードを使用している場合:
  - Layout: `Japanese`
  - Variant: `Japanese` (または `Japanese (OADG 109A)`)
- `Done` を選択して次へ進みます。

### 5.3 インストールタイプの選択

> **ここが最重要の選択です。**

- 「**Ubuntu Server (minimized)**」を選択します。
  - 通常の `Ubuntu Server` との違いは、不要なパッケージが最初から排除されており、サーバー用途に最適化されています。

### 5.4 ネットワーク設定
- 有線LANを接続していれば、DHCPで自動的にIPアドレスが割り当てられます。
- IPアドレスが表示されていれば問題ありません。
- `Done` で次へ進みます。

> **ℹ️ 補足:** サーバーの固定IPはOS側でなくルーター（DHCP予約）で設定することを推奨します。OSのnetplan設定は変更不要です。

### 5.5 プロキシ設定
- 社内プロキシがなければ空白のまま `Done` で次へ進みます。

### 5.6 Ubuntuアーカイブミラー
- 日本のミラーサーバーが自動検出されます（例: `jp.archive.ubuntu.com`）。
- そのまま `Done` で次へ進みます。

### 5.7 ストレージ設定（パーティション）
- 「**Use an entire disk**」を選択します（推奨）。
- 「Set up this disk as an LVM group」のチェックは **外す** ことを推奨します。
  - LVMはボリューム管理が複雑になるため、シンプル構成のサーバーには不要です。
- インストール先のディスクを確認して `Done` を選択します。
- 確認ダイアログで「**Continue**」を選択します（ディスクが初期化されます）。

### 5.8 ユーザーアカウントの設定

#### Your name（表示名）
- SSH接続時やログイン後のプロンプトには**表示されない**、飾りの名前です。
- 実用上ほぼ使われないため、何を入れても構いません。**本名である必要はありません。**
- 例: `admin`、`Server Owner`、空白でもOK

#### Your server's name（ホスト名）
- SSH接続中のターミナルプロンプト（例: `kenta@homeserver:~$`）に表示されます。
- このサーバーが何のサーバーかわかる、**短くシンプルな英数字**にしましょう。
- 使用可能な文字: 英小文字・数字・ハイフン（スペース・日本語・アンダースコアは不可）
- 例: `homeserver`、`gpu-server`、`nas01`

#### Pick a username（ログインユーザー名）
- **最も重要な項目です。** SSHログイン時（`ssh kenta@192.168.x.x`）や `sudo` コマンドで毎回使います。
- **短くて打ちやすい英小文字**にすることを強く推奨します。
- 使用可能な文字: 英小文字・数字・アンダースコア・ハイフン（スペース・大文字・日本語は不可）
- 例: `kenta`、`admin`、`user01`

#### Password（パスワード）
- 推測されにくい十分な強度のパスワードを設定してください。
- Tailscale/SSHキー認証を使う場合でも、`sudo` 実行時に必要になります。

> **⚠️ 重要:** ユーザー名とホスト名はあとから変更が面倒です。慎重に決定してください。

### 5.9 Ubuntu Pro (スキップ)
- 「Skip for now」を選択して `Done` で次へ進みます。
- 個人利用の場合は不要です。

### 5.10 OpenSSH Server のインストール

> **⚠️ 重要:** 必ずチェックを入れてください。

- 「**Install OpenSSH server**」に `Space` キーでチェックを入れます（`[X]` になれば有効）。
- これにより、インストール完了後すぐにSSHでリモート接続できるようになります。

### 5.11 追加パッケージ（Featured Server Snaps）
- 何も選択せずにそのまま `Done` で次へ進みます。
- DockerなどはSnapではなく、公式の手順（§3）でインストールするため不要です。

---

## Step 6: インストールの実行と完了

1. これまでの設定を確認後、インストールが開始されます（5〜15分程度）。
2. インストール完了後、「**Install complete!**」のメッセージと「**Reboot Now**」ボタンが表示されます。
3. 「**Reboot Now**」を選択します。
4. 「Please remove the installation medium, then press ENTER」と表示されたら、**USBメモリを抜いて** `Enter` を押します。

---

## Step 7: 初回ログインの確認

### サーバー本体から直接ログイン
再起動後、ログインプロンプトが表示されます。
```
homeserver login: kenta
Password: (設定したパスワードを入力)
```
ログインできれば成功です。

### WindowsからSSHで接続（推奨）

1. **サーバーのIPアドレスを確認する:**
   ```bash
   # サーバー上で実行
   ip addr show | grep "inet "
   # または
   hostname -I
   ```

2. **WindowsのPowerShell（またはターミナル）から接続:**
   ```powershell
   ssh kenta@192.168.x.x
   ```
   - `kenta` はインストール時に設定したユーザー名
   - `192.168.x.x` はサーバーのIPアドレス

3. 初回接続時に「接続しますか？(yes/no)」と聞かれたら `yes` と入力します。

---

## 完了後の次のステップ

インストールが完了したら `ubuntu_setup_flow.md` の **§1.2 システムの最新化と基本ツールの導入** に進んでください。

```bash
sudo apt update && sudo apt upgrade -y
```
